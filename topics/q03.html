Question 3

Consider the following regular expression meta operators:

() [] {} . * + ? ^ $ \ |

For each of the following, give a couple of examples of strings which the regular expression would match. Describe (colloquially, in a manner that a non-technical person would understand) the set of strings that the pattern is designed to match. Each regular expression is enclosed in the pair of '/' characters below. [2 marks each]

(a) Regular Expression: /^ [A-Za-z][a-z]* $/

Examples of matching strings:

  • "Apple"
  • "Screen"
  • "Exam"

Explanation: This regular expression matches any string that starts with an uppercase or lowercase letter (from A to Z or a to z) and is followed by zero or more lowercase letters. For instance, "Apple" or "Screen" match because they start with a capital letter followed by lowercase letters. The pattern does not match words like "apple" if they do not start with a capital letter. It is designed to match single words that have an initial uppercase letter, commonly used for proper nouns or titles.

(b) Regular Expression: /\s(\w+)\s\1/

Examples of matching strings:

  • " hello hello "
  • " hi hi "
  • " comp20008 comp20008 "

Explanation: This regular expression matches a space followed by any word (letters, numbers, or underscores) and then a space followed by the same word again. For example, "hello hello" or "hi hi" match this pattern. The idea is that the word appears twice consecutively, separated by a space. This could be used to detect repeated words in a text, which is useful for identifying errors or repetitive phrasing.

Practice Question 1

Regular Expression: /^[0-9]{4}$/

Give examples of strings that match the pattern and explain what the pattern is designed to match.

Examples of matching strings:

  • "2023"
  • "1234"

Explanation: This regular expression matches any string that contains exactly four digits. For example, "2023" and "1234" match because they are composed of exactly four numerical characters. This pattern is useful for matching years, PIN codes, or any numeric value with precisely four digits.

Practice Question 2

Regular Expression: /^[A-Za-z0-9_]+$/

Give examples of strings that match the pattern and explain what the pattern is designed to match.

Examples of matching strings:

  • "user_name1"
  • "Hello123"

Explanation: This regular expression matches any string that consists of uppercase letters, lowercase letters, digits, or underscores. It matches strings like "user_name1" and "Hello123" but does not match strings containing special characters or spaces. This is commonly used to validate usernames or identifiers in programming.

Practice Question 3

Regular Expression: /^(\d{1,3}\.){3}\d{1,3}$/

Give examples of strings that match the pattern and explain what the pattern is designed to match.

Examples of matching strings:

  • "192.168.1.1"
  • "127.0.0.1"

Explanation: This regular expression matches an IP address in IPv4 format. It matches strings like "192.168.1.1" or "127.0.0.1", which consist of four groups of one to three digits, each group separated by a dot. This pattern is useful for validating IP addresses in network programming or web development.

Summary

Regular expressions are powerful tools for pattern matching and text processing. Understanding the various meta operators and how they work can help you build complex patterns to match specific types of strings. Practice identifying the patterns and understanding their use cases to get more comfortable with regular expressions.
