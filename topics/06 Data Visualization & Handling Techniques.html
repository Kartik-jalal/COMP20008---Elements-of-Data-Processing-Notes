COMP20008 - Data Visualization & Handling Techniques: Exam Notes

Scatter Plots & Advanced Scatter Plot Techniques

  • Scatter Plots: Used to visualize the relationship between two numerical variables.
  • Bubble Plots: Represent three-dimensional data by varying the size of data points.
  • Enhanced Scatter Plots: Use color to represent a third variable, adding more depth to the data representation.

Heat Maps

  • Definition: Visualize a data matrix where individual values are represented using colors.
  • Use Case: Often used when data objects are sorted by class or type to show differences.
  • Normalization: Normalizing features is crucial to prevent one feature from dominating the visualization.

Axes Scaling & Parallel Coordinates

  • Axes Scaling: Scaling features helps achieve better visual balance, especially in parallel coordinates.
  • Parallel Coordinates: Used to visualize multivariate data; the ordering of axes can reveal specific correlations between features.

Handling Missing Data

  • Types of Missing Data:
    • MCAR: Missing Completely at Random - missingness is unrelated to any data.
    • MAR: Missing at Random - missingness is related to observed data.
    • MNAR: Missing Not at Random - missingness is related to the missing data itself.

Disguised Missing Data

  • Definition: Missing data that appears as suspicious or placeholder values (e.g., "xx@xx.com").
  • Example: Incorrect postal code used in a rental system due to an incompatible field.
  • Handling: Use domain knowledge to identify and rectify unusual values.

Relational Databases & SQL

  • Relational Database: Stores data in tables linked by relationships (e.g., student records).
  • SQL: Structured Query Language, used to interact with relational databases to retrieve, update, and manage data.

Semi-Structured Data

  • CSV: Comma-separated values, used for storing tabular information in a simple text format.
  • HTML: Markup language used to define content structure, often considered semi-structured data.

XML Syntax & CDATA

  • XML Syntax: XML preserves white spaces and uses special characters like '<' and '&'.
  • CDATA: CDATA sections can be used to include large blocks of text containing special characters.
  • Example: To include '&' in XML, use '&' or wrap the text in CDATA.

JSON vs. XML

  • JSON: A lightweight data-interchange format, simpler and easier to parse compared to XML.
  • Python Libraries: Libraries like json and lxml can be used to handle JSON and XML data.
  • Example: JSON is often used in web APIs due to its simplicity.

PDF Data Extraction

  • Tools: Libraries like PDFMiner can be used to extract text from PDFs.
  • Challenge: Converting PDFs to text may result in the loss of formatting, making data extraction challenging.
  • Example: Extracting tables from PDFs often requires additional processing to retain the structure.

String Similarity & Edit Distance

  • Edit Distance: Measures the number of operations required to transform one string into another (e.g., insert, delete, replace).
  • Levenshtein Distance: A common edit distance metric used for comparing strings.
  • Example: Transforming "kitten" to "sitting" requires three edits: replace 'k' with 's', add 'i', and add 'g'.

N-gram Distance

  • N-grams: Substrings of length 'n' used for approximate string matching.
  • Use Case: N-grams are useful for detecting similarities between text by comparing overlapping sequences.
  • Example: Bi-grams of the word "data" are "da", "at", and "ta".

Pattern Matching with Regular Expressions

  • Regular Expressions (Regex): Used to search and manipulate text based on patterns.
  • Anchoring: Use '^' to match the start of a string and '$' to match the end.
  • Example: The pattern '^data' will match any string that starts with "data".
