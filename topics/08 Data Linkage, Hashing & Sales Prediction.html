Blocking Techniques for Data Linkage

Blocking is used to reduce the number of comparisons needed for record linkage:

  • Blocking by Release Year: Grouping movies by release year (e.g., "2016").
  • Blocking by Concatenation: Concatenating attributes like "release year" and a title word (e.g., "2016 ghost").
  • Handling Redundancies: Using multiple functions (e.g., year, year+1, year-1) can lead to redundant blocks.

Example: A movie record "ghost busters, 2016" may be blocked by "2015", "2016", and "2017" to handle potential mismatches.

  • Handling Conflicting Attributes: Merging records may require resolving conflicts, such as different values for the same attribute.
  • False Positives & Negatives: Matching systems may still produce errors.

Example: Two movie records may be matched if their similarity score is above 0.5, but manual inspection may still be needed for confirmation.

Hash Functions

Hashing is used to map data items of arbitrary size to a fixed size:

  • Hash Function (H): A function that takes an input and returns a fixed-size value.
    H(X) = hashed value of X
  • Non-Invertible Hash Functions: These are one-way functions, meaning that given H(X), it is extremely hard to reconstruct X.
    • Example: MD5 hash function produces a 32-digit hexadecimal hash.

Privacy-Preserving Exact Match Schemes

  • Trusted Third Party: Organisations A and B use a trusted third party (C) to determine matches between records.
  • Hashing with Salt: One-way hash functions with a salt are used to increase privacy.
  • Adding Random Records: Dummy records are added to prevent frequency attacks.

Example: Organisations A and B hash their records with a shared salt and add dummy records before sending them to C for matching.

Bloom Filters in Record Linkage

Bloom filters are used to represent sets for privacy-preserving record linkage:

  • Bit Array Length (l) and Hash Functions (k): A and B agree on these parameters to create Bloom filters.
  • Dummy Records: Added to reduce the chances of a frequency attack by the third party.
  • Challenges: Bloom filters are not very secure for short strings due to collisions.
